# README Template

Use this when the repository needs a short, reader-facing entrypoint.

```md
# <Project Name>

<One short paragraph: what the project is and what outcome it is trying to produce.>

Current documents:

- `<path>` - <what this document defines>
- `<path>` - <what this document defines>
- `<path>` - <what this document defines>

Optional local development:

- `<command>` - <what it does>
- `<command>` - <what it does>
```

## Writing guidance

- Keep the README compact.
- Link to the whitepaper and other foundational docs instead of pasting them again.
- Only include commands that are already real and useful.
- If the project is still early, say that plainly instead of pretending the repo is feature-complete.
